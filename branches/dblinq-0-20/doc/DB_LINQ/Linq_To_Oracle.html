DbLinq Project: Linq Provider for Oracle

Oracle-Specific Notes:

1. The supported version of Oracle is Oracle XE.
2. For Oracle, Live object cache does not work yet (it does work in the Postgres and Mysql flavors of DbLinq, though)
3. Since Oracle does not support Identity columns, I am planning to use naming convention to lookup sequence names (eg. "myTable_seq" for table "myTable"). This is in the same spirit as Ruby on Rails.

Getting Started:

1. Download general DbLinq required items, listed on the main project page
2. Download Postgres 8.2 
3. Download DbLinq.Oracle.zip
4. run OracleMetal.exe against your existing database to create a schema .cs file
5. Add the cs file and DbLinq.Oracle.Prototype.dll to your .NET project
